# S0 — Fundação multi-tenant Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Estabelecer a base multi-tenant — schema `campanha`, `audit_log` imutável, RLS de isolamento, PostGIS e view pública de subdomínio — direto no projeto Supabase na nuvem via MCP, mais o middleware de subdomínio no scaffold Next.

**Architecture:** Banco único Postgres (Supabase cloud) com isolamento por RLS sobre `campanha_id`. Migrations aplicadas via MCP `apply_migration` e copiadas para `supabase/migrations/` no repo. Superadmin opera via `service_role` (ignora RLS) num backend server-side separado — sem policy de exceção no banco. Subdomínio resolvido por middleware Next contra uma view pública não-PII; a barreira real é o RLS.

**Tech Stack:** Next.js (App Router, TypeScript), Supabase cloud (Postgres 17, PostGIS), `@supabase/supabase-js`, Vitest (testes unitários do middleware), MCP Supabase (`apply_migration`, `execute_sql`, `get_advisors`).

## Global Constraints

- **Projeto Supabase cloud:** `axcftjqdjvknrpqzrxls` (org `hzucdcptinqcgepwvpih`), região `sa-east-1` (São Paulo). Toda chamada MCP usa esse `project_id`. (ADR 0010 — residência Brasil.)
- **RLS default-deny:** RLS ligado em toda tabela operacional; nenhuma policy permissiva sobrando. (ADR 0001/0004.)
- **Contrato de claim (fixado aqui, preenchido no S1):** `app_metadata.campanha_id` (uuid) e `app_metadata.papel` (text). Policies leem `auth.jwt() -> 'app_metadata' ->> 'campanha_id'`.
- **Audit append-only:** `audit_log` sem UPDATE/DELETE para `authenticated`/`anon`. Nem Gestor nem Superadmin editam. (ADR 0014.)
- **Sem exceção Superadmin no banco:** nenhuma policy `is_superadmin`; Superadmin só via `service_role` server-side. (Decisão aprovada.)
- **Sem extensão de cripto no S0:** HMAC do índice cego (CPF/título) fica para o S2, chave fora do banco. (ADR 0010.)
- **Cópia versionada:** toda migration aplicada via MCP é salva idêntica em `supabase/migrations/NNNN_<slug>.sql`.

create table public.audit_log (
  id bigint generated always as identity primary key,
  campanha_id uuid not null references public.campanha(id),
  actor_id uuid,
  acao text not null,
  entidade text,
  entidade_id text,

-- Isolamento por campanha via claim (preenchida pelo hook do S1).

  with check ( campanha_id = (auth.jwt() -> 'app_metadata' ->> 'campanha_id')::uuid );

// Domínios raiz conhecidos (sem subdomínio de campanha). Em produção é o
// domínio do produto; em dev, localhost.
const ROOT_HOSTS = new Set(['dominio.com.br', 'localhost']);
const IGNORED = new Set(['www']);

export function extractSubdomain(host: string): string | null {
  const hostname = host.split(':')[0]; // tira a porta
  if (ROOT_HOSTS.has(hostname)) return null;

  const parts = hostname.split('.');
  // localhost com subdomínio: "campanha-a.localhost"
  if (parts.length === 2 && parts[1] === 'localhost') {
    return IGNORED.has(parts[0]) ? null : parts[0];
  }
  // domínio real: precisa de subdomínio + domínio + ao menos 1 TLD parte
  if (parts.length < 3) return null;
  const sub = parts[0];
  return IGNORED.has(sub) ? null : sub;
}

- [ ] **Step 6: Escrever o middleware**

Create `web/middleware.ts`:
```ts
import { NextRequest, NextResponse } from 'next/server';
import { extractSubdomain } from './lib/subdomain';
import { publicClient } from './lib/supabase/server';

export const config = {
  // Ignora assets e a própria rota de saúde.
  matcher: ['/((?!_next/static|_next/image|favicon.ico|health).*)'],
};

export async function middleware(req: NextRequest) {
  const host = req.headers.get('host') ?? '';
  const subdominio = extractSubdomain(host);

  // Sem subdomínio (domínio raiz) → segue sem campanha selecionada.
  if (!subdominio) return NextResponse.next();

  const supabase = publicClient();
  const { data } = await supabase
    .from('campanha_publica')
    .select('subdominio, nome, status')
    .eq('subdominio', subdominio)
    .maybeSingle();

  // Subdomínio inexistente → 404.
  if (!data) {
    return new NextResponse('Campanha não encontrada', { status: 404 });
  }
  // Campanha não ativa (suspensa/encerrada) → bloqueio.
  if (data.status !== 'ativa') {
    return new NextResponse('Campanha indisponível', { status: 403 });
  }

  const res = NextResponse.next();
  res.headers.set('x-campanha-subdominio', subdominio);
  return res;
}
```
